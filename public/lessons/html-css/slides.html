🌐 Qu'est-ce que le HTML ?

HTML = HyperText Markup Language

Le langage de balisage standard pour créer des pages web

🎯 Rôle : Définir la structure et le contenu (pas l'apparence)

🏗️ Structure de base

<!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Ma première page</title>
</head>
<body>
    <h1>Bonjour le monde !</h1>
    <p>Ceci est ma première page HTML.</p>
</body>
</html>

🔖 Anatomie d'une balise HTML

<h1>Mon titre</h1>

🏷️ Balise ouvrante

<h1> - Début de l'élément

📝 Contenu

Le texte entre les balises

🏷️ Balise fermante

</h1> - Fin de l'élément

⚡ Balises auto-fermantes

<br> <hr> <img>

📋 Les éléments essentiels

🔖 DOCTYPE

Déclare le type de document HTML5

🌐 html

Élément racine, contient tout

🧠 head

Métadonnées invisibles

👁️ body

Contenu visible de la page

🧠 Balises HEAD essentielles

<head>
    <!-- Encodage des caractères (OBLIGATOIRE) -->
    <meta charset="UTF-8">
    
    <!-- Responsive design (OBLIGATOIRE pour mobile) -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    
    <!-- Titre de la page (affiché dans l'onglet) -->
    <title>Titre de ma page</title>
    
    <!-- SEO - Description pour Google -->
    <meta name="description" content="Description de la page pour les moteurs de recherche">
    
    <!-- SEO - Mots-clés -->
    <meta name="keywords" content="html, css, web, développement">
    
    <!-- Auteur -->
    <meta name="author" content="Votre nom">
</head>

🎯 Pour vos exercices : Utilisez toujours charset UTF-8 et viewport pour le responsive

📊 Hiérarchie des titres

h2 - Titre de section

h3 - Sous-titre

h4 - Titre mineur

🎯 Règles d'or :

  • Un seul <h1> par page
  • Hiérarchie logique : h1 → h2 → h3 (ne pas sauter de niveaux)
  • Les titres structurent le contenu, pas l'apparence

📝 Exercice 04 : Jeu de Correction

🎮 Mission Détective

Corrigez toutes les erreurs dans ce code HTML mal structuré

🔍 Erreurs à identifier

❌ Problèmes structurels

  • Hiérarchie des titres incorrecte (h3 au lieu de h2)
  • Titre sans majuscules
  • Manque de structure claire
  • Informations mélangées

❌ Problèmes de formatage

  • Aucun formatage du texte (strong, em)
  • Date mal placée et mal formatée
  • Manque de détails (ingrédients, étapes)
  • Pas de séparateurs visuels

🔧 Corrections à apporter

h3 → h2
Corriger hiérarchie
<strong>
Mots importants
<em>
Temperatures, temps
<small>
Date publication
<hr>
Séparer sections
<mark>
Astuces importantes

🕵️ Stratégie de correction :

  1. Corriger la hiérarchie des titres en premier
  2. Ajouter les majuscules et la ponctuation
  3. Structurer en sections logiques
  4. Ajouter les détails manquants (ingrédients précis)
  5. Formater avec strong, em, mark
  6. Placer correctement les métadonnées

🤔 Pourquoi utiliser des listes ?

🎯 Les listes permettent de :

📊 Structurer l'information

  • Organiser des éléments similaires
  • Créer une hiérarchie claire
  • Améliorer la lisibilité

♿ Améliorer l'accessibilité

  • Lecteurs d'écran comprennent mieux
  • Navigation clavier facilitée
  • Sémantique claire pour les robots

🎯 Règle d'or : Utilisez des listes dès que vous avez 2 éléments ou plus du même type !

📝 Les 3 types de listes HTML

📌 Liste non ordonnée

<ul>

Pour des éléments sans ordre spécifique

• Pommes
• Bananes
• Oranges

🔢 Liste ordonnée

<ol>

Pour des étapes ou un ordre important

1. Première étape
2. Deuxième étape
3. Troisième étape

📖 Liste de définition

<dl>

Pour des termes et leurs définitions

HTML
  Markup Language
CSS
  Style Sheets

📌 Liste non ordonnée (ul)

<!-- Structure de base -->
<ul>
  <li>Premier élément</li>
  <li>Deuxième élément</li>
  <li>Troisième élément</li>
</ul>

<!-- Avec formatage -->
<ul>
  <li><strong>Pommes</strong> (1 kg)</li>
  <li><em>Bananes</em> bio (6 pièces)</li>
  <li>Oranges <small>(origine Espagne)</small></li>
</ul>

Résultat visuel :

  • Premier élément
  • Deuxième élément
  • Troisième élément

Avec formatage :

  • Pommes (1 kg)
  • Bananes bio (6 pièces)
  • Oranges (origine Espagne)

💡 Utilisez ul pour : Ingrédients, fonctionnalités, avantages, éléments de navigation

🔢 Liste ordonnée (ol)

<!-- Structure de base -->
<ol>
  <li>Première étape</li>
  <li>Deuxième étape</li>
  <li>Troisième étape</li>
</ol>

<!-- Avec numérotation personnalisée -->
<ol start="5">
  <li>Commence à 5</li>
  <li>Puis 6</li>
  <li>Et 7</li>
</ol>

Résultat visuel :

  1. Première étape
  2. Deuxième étape
  3. Troisième étape

Numérotation personnalisée :

  1. Commence à 5
  2. Puis 6
  3. Et 7

💡 Utilisez ol pour : Recettes, instructions, étapes, classements, chronologie

⚙️ Attributs des listes ordonnées

<!-- Types de numérotation -->
<ol type="1"><!-- 1, 2, 3 (défaut) -->
<ol type="A"><!-- A, B, C -->
<ol type="a"><!-- a, b, c -->
<ol type="I"><!-- I, II, III -->
<ol type="i"><!-- i, ii, iii -->

<!-- Numérotation personnalisée -->
<ol start="10"><!-- Commence à 10 -->

<!-- Ordre inversé -->
<ol reversed><!-- Compte à rebours -->

Type="A" (majuscules) :

  1. Première option
  2. Deuxième option
  3. Troisième option

Type="i" (chiffres romains minuscules) :

  1. Point principal
  2. Point secondaire
  3. Point tertiaire

🎯 Cas d'usage :

  • type="A" : Sections de documents (A. Introduction, B. Développement)
  • type="i" : Sous-sections (i. Définition, ii. Exemples)
  • start="X" : Continuer une numérotation
  • reversed : Classements (Top 10 → Top 1)

📖 Liste de définition (dl)

<!-- Structure de base -->
<dl>
  <dt>Terme à définir</dt>
  <dd>Définition du terme</dd>
  
  <dt>Autre terme</dt>
  <dd>Sa définition</dd>
</dl>

<!-- Exemple concret -->
<dl>
  <dt><strong>HTML</strong></dt>
  <dd>HyperText Markup Language</dd>
  
  <dt><strong>CSS</strong></dt>
  <dd>Cascading Style Sheets</dd>
</dl>

Résultat visuel :

HTML
HyperText Markup Language - Langage de balisage pour créer des pages web
CSS
Cascading Style Sheets - Langage de style pour la présentation
JavaScript
Langage de programmation pour l'interactivité web

💡 Utilisez dl pour : Glossaires, FAQ, spécifications techniques, contact info

🔄 Listes imbriquées

<!-- Liste dans liste -->
<ol>
  <li>Préparation du voyage
    <ul>
      <li>Réserver les vols</li>
      <li>Choisir l'hébergement</li>
      <li>Faire les valises</li>
    </ul>
  </li>
  <li>Pendant le voyage
    <ul>
      <li>Prendre des photos</li>
      <li>Goûter la cuisine locale</li>
    </ul>
  </li>
  <li>Retour de voyage</li>
</ol>

Résultat visuel :

  1. Préparation du voyage
    • Réserver les vols
    • Choisir l'hébergement
    • Faire les valises
  2. Pendant le voyage
    • Prendre des photos
    • Goûter la cuisine locale
  3. Retour de voyage

🎯 Bonnes pratiques pour l'imbrication :

  • Maximum 3 niveaux de profondeur
  • Mélangez ol et ul selon le contexte
  • Indentez correctement votre code
  • Fermez toujours les balises dans l'ordre

🧰 Toutes les balises des listes

<ul>
Unordered List - Liste non ordonnée
<ol>
Ordered List - Liste ordonnée
<li>
List Item - Élément de liste
<dl>
Definition List - Liste de définition
<dt>
Definition Term - Terme à définir
<dd>
Definition Description - Définition

🔧 Attributs importants :

  • type="1|A|a|I|i" sur ol - Type de numérotation
  • start="5" sur ol - Numéro de départ
  • reversed sur ol - Ordre inversé
  • value="10" sur li - Valeur spécifique d'un item

<!-- ERREUR: li en dehors de ul/ol -->
<li>Élément isolé</li>

<!-- ERREUR: Mauvais type de liste -->
<ul>
  <li>Étape 1</li>
  <li>Étape 2</li>
</ul>

<!-- ERREUR: Imbrication trop profonde -->
<ul>
  <li>Niveau 1
    <ul>
      <li>Niveau 2
        <ul>
          <li>Niveau 3
            <ul>
              <li>Niveau 4 (trop!)</li>

✅ Code correct

<!-- CORRECT: li dans ul -->
<ul>
  <li>Élément dans sa liste</li>
</ul>

<!-- CORRECT: ol pour les étapes -->
<ol>
  <li>Première étape</li>
  <li>Deuxième étape</li>
</ol>

<!-- CORRECT: Maximum 3 niveaux -->
<ol>
  <li>Niveau 1
    <ul>
      <li>Niveau 2
        <ul>
          <li>Niveau 3 (max)</li>

🔧 Conseil de debugging : Validez votre HTML avec le validateur W3C pour détecter les erreurs de structure !
